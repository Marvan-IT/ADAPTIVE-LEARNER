### 2.3 JSONB Shape

**Translation column shape (all five columns use the same shape):**

```json
{
  "en_source_hash": "a3f9d2c1b0e4...",
  "ml": "ആദ്യ ഗണിതശാസ്ത്രം",
  "ta": "ஆரம்ப கணிதம்",
  "ar": "الرياضيات الأساسية",
  "hi": "बुनियादी गणित",
  "fr": "Mathématiques de base",
  "es": "Matemáticas básicas",
  "zh": "基础数学",
  "ja": "基礎数学",
  "de": "Grundlegende Mathematik",
  "ko": "기초 수학",
  "pt": "Matemática básica",
  "si": "මූලික ගණිතය"
}
```

- `en_source_hash`: SHA-1 hex digest of the English source string (used for idempotency on re-runs).
- Language entries are present only for languages that have been successfully translated. Missing entry = English fallback applies.
- `en` is never stored as a translation entry (it is always the English column itself).

### 2.4 `presentation_text` JSON Shape

**Before (current flat shape):**

```json
{
  "cache_version": 3,
  "cards": [...],
  "concepts_queue": [...],
  "concepts_covered": [...]
}
```

**After (per-language shape):**

```json
{
  "cache_version": 3,
  "by_language": {
    "en": {
      "presentation": "...",
      "cards": [...],
      "concepts_queue": [...],
      "concepts_covered": [...]
    },
    "ml": {
      "presentation": "...",
      "cards": [...],
      "concepts_queue": [...],
      "concepts_covered": [...]
    }
  }
}
```

**Legacy-shape adapter algorithm** (executed in `CacheAccessor._ensure_by_language_shape()`):

```
1. raw = json.loads(presentation_text)  # or {}
2. If "by_language" in raw: return raw  # already new shape, nothing to do
3. If raw is empty or has no usable content: return {"cache_version": -1, "by_language": {}}
4. # Flat legacy shape detected
5. lang = student.preferred_language or "en"
6. slice_data = {
     "presentation": raw.get("presentation", ""),
     "cards": raw.get("cards", []),
     "concepts_queue": raw.get("concepts_queue", []),
     "concepts_covered": raw.get("concepts_covered", []),
   }
7. new_shape = {
     "cache_version": raw.get("cache_version", -1),
     "by_language": {lang: slice_data}
   }
8. persist new_shape back to session.presentation_text
9. return new_shape
```

If `cache_version` in the legacy shape is -1, step 3's alternative branch handles it: wrapped as `{"cache_version": -1, "by_language": {}}`, which forces regeneration on the next card fetch.

**LRU eviction rule (enforced in `CacheAccessor.set_language_slice()`):**

- Max 5 language entries per `by_language` dict.
- If adding a new language would exceed 5: remove the entry with the fewest `len(cards)` (proxy for "least complete" rather than "oldest" — simplest without timestamps).
- If the `len(json.dumps(new_shape))` would exceed 512 000 bytes after the write: evict until it fits; if it still doesn't fit after removing all but the target language, truncate `cards` to 5 per slice (fallback — log warning `session_cache_truncated`).

### 2.5 Caching Strategy

| Layer | Cache | TTL / Invalidation |
|-------|-------|--------------------|
| JSONB translation columns | PostgreSQL row | Invalidated only when English source changes (hash mismatch on re-run) |
| `presentation_text` per-language slice | PostgreSQL `Text` column | Invalidated when `cache_version` is bumped (cross-language; same as today) |
| `_title_translation_cache` (disk JSON) | `backend/translation_cache.json` | **Retired** in Phase 4; seeded into DB in Phase 3 and then deleted |
| In-memory graph nodes | `ChunkKnowledgeService` (in-memory) | Reload on restart; translation overlay applied per-request, not cached |

### 2.6 Data Retention

- Completed sessions: `presentation_text` purged (set to NULL) after 7 days.
- Abandoned sessions (no `completed_at`, `started_at` > 90 days ago): full row deletion.
- Both are daily cron jobs; to be implemented by devops-engineer agent in Phase 5.

```sql
SELECT
  ts.id,
  ts.concept_id,
  ts.book_slug,
  ts.phase,
  ts.check_score,
  ts.concept_mastered,
  ts.started_at,
  ts.completed_at,
  -- Take the first (lowest order_index) chunk heading for this concept as title
  COALESCE(
    (cc.heading_translations -> :lang)::text,
    cc.heading,
    ts.concept_id
  ) AS concept_title,
  COALESCE(
    (b.title_translations -> :lang)::text,
    b.title,
    ts.book_slug
  ) AS book_title
FROM teaching_sessions ts
LEFT JOIN LATERAL (
  SELECT heading, heading_translations
  FROM concept_chunks
  WHERE book_slug = ts.book_slug
    AND concept_id = ts.concept_id
  ORDER BY order_index ASC
  LIMIT 1
) cc ON true
LEFT JOIN books b ON b.book_slug = ts.book_slug
WHERE ts.student_id = :student_id
  AND ts.phase IN ('PRESENTING','CARDS','CARDS_DONE','SELECTING_CHUNK','CHECKING','COMPLETED')
ORDER BY ts.started_at DESC
LIMIT 50
```

`lang` = `student.preferred_language`.

The `LATERAL` join is PostgreSQL-native and efficient with the existing `ix_concept_chunks_book_concept_order` index.

### 3.4 `PATCH /students/{id}/language`

**No request/response shape change.** Internal behavior change only.

**Current behavior:** calls `_translate_summaries_headings()` (LLM, 2–5 s).

**New behavior:**

```sql
SELECT cc.id, cc.heading, cc.heading_translations
FROM concept_chunks cc
WHERE cc.book_slug = :book_slug
  AND cc.concept_id = :concept_id
ORDER BY cc.order_index ASC
```

Then apply `resolve_translation(cc.heading, cc.heading_translations, req.language)` for each row and return the list as `translated_headings`.

**Session cache change:** Instead of wiping `cards = []`, set a lazy-regen marker:

```python
cache = CacheAccessor.ensure_by_language_shape(active_session.presentation_text, student)
# Don't delete the new language's slice if it already exists.
# Only reset cache_version to -1 for the target language slice if it's already cached
# (forces regeneration on next fetch for that language).
# This preserves the *other* language slices intact.
if req.language in cache["by_language"]:
    cache["by_language"][req.language]["cache_version"] = -1
# Write back
active_session.presentation_text = json.dumps(cache)
session_cache_cleared = True
```

`session_cache_cleared` remains `True` so the frontend knows to call `reloadCurrentChunk`.

**Target:** p95 < 100 ms. The `PATCH` now does: 1 UPDATE on `students`, 1 SELECT on `concept_chunks` (indexed by `book_slug + concept_id`), 1 JSON parse/serialize, 1 UPDATE on `teaching_sessions`. All within one DB transaction.

## 5. Python Helper Function Signatures

### 5.1 `get_request_language()`

**File:** `backend/src/api/dependencies.py` (new file)

```python
from fastapi import Request
from db.models import Student
from api.prompts import LANGUAGE_NAMES

SUPPORTED_LANG_CODES: frozenset[str] = frozenset(LANGUAGE_NAMES.keys())


async def get_request_language(
    request: Request,
    student: Student | None = None,
) -> str:
    """
    Resolve the request language.
    
    Priority:
    1. student.preferred_language (for authenticated endpoints with a loaded student)
    2. Accept-Language header base tag (for catalog / unauthenticated endpoints)
    3. "en" (default fallback)
    
    Always returns a code present in SUPPORTED_LANG_CODES.
    """
    if student and student.preferred_language:
        lang = student.preferred_language.strip().lower()
        return lang if lang in SUPPORTED_LANG_CODES else "en"
    header = request.headers.get("Accept-Language", "en")
    base = header.split(",")[0].split(";")[0].split("-")[0].strip().lower()
    return base if base in SUPPORTED_LANG_CODES else "en"
```

### 5.2 `resolve_translation()`

**File:** `backend/src/api/dependencies.py`

```python
def resolve_translation(
    english_value: str,
    translations: dict,
    lang: str,
) -> str:
    """
    Return the translated value for lang, falling back to english_value.
    
    Args:
        english_value: The English column value (never None in the schema).
        translations: The JSONB column value (dict). May be {} for untranslated content.
        lang: Target language code.
    
    Returns:
        Non-empty string. Never the slug, never empty.
    
    Note: 'en_source_hash' key in translations is skipped automatically
    since it is not a language code.
    """
    if lang == "en" or not translations:
        return english_value
    translated = translations.get(lang)
    if translated and isinstance(translated, str) and translated.strip():
        return translated
    return english_value
```

### 5.3 `CacheAccessor` class

**File:** `backend/src/api/teaching_service.py` (new inner class, used in `TeachingService`)

```python
import hashlib
import json
import logging

_CACHE_LOGGER = logging.getLogger("teaching_service.cache")

CACHE_MAX_LANGUAGES = 5       # LRU cap: max language slices per session
CACHE_MAX_BYTES     = 512_000 # Hard limit per presentation_text row


class CacheAccessor:
    """
    Read/write per-language slices of TeachingSession.presentation_text.
    Thread-safety: each request owns its own session object; no shared state.
    """

    @staticmethod
    def ensure_by_language_shape(
        raw_text: str | None,
        student_lang: str,
    ) -> dict:
        """
        Parse presentation_text and ensure it is in the by_language shape.
        Wraps legacy flat-shape JSON on first read.
        
        Returns the mutable dict. Caller must persist the dict back to the
        session row if the shape was upgraded.
        """
        from json_repair import repair_json  # already a project dependency
        
        if not raw_text:
            return {"cache_version": -1, "by_language": {}}
        
        try:
            raw = json.loads(raw_text)
        except json.JSONDecodeError:
            try:
                raw = json.loads(repair_json(raw_text))
            except Exception:
                return {"cache_version": -1, "by_language": {}}
        
        if "by_language" in raw:
            return raw  # already new shape
        
        # Legacy flat shape — wrap it
        cache_version = raw.get("cache_version", -1)
        slice_data = {
            "presentation": raw.get("presentation", ""),
            "cards": raw.get("cards", []),
            "concepts_queue": raw.get("concepts_queue", []),
            "concepts_covered": raw.get("concepts_covered", []),
        }
        new_shape = {
            "cache_version": cache_version,
            "by_language": {student_lang: slice_data},
        }
        _CACHE_LOGGER.info(
            "[cache] Upgraded legacy presentation_text to by_language shape for lang=%s",
            student_lang,
        )
        return new_shape

    @staticmethod
    def get_language_slice(cache: dict, lang: str) -> dict | None:
        """
        Return the language slice dict or None (cache miss).
        Only returns a slice if cache_version >= 0 (version -1 = stale).
        """
        if cache.get("cache_version", -1) < 0:
            return None
        return cache.get("by_language", {}).get(lang)

    @staticmethod
    def set_language_slice(cache: dict, lang: str, slice_data: dict) -> dict:
        """
        Write a language slice. Enforces CACHE_MAX_LANGUAGES LRU eviction
        and CACHE_MAX_BYTES size cap.
        
        Returns the (possibly mutated) cache dict.
        """
        by_lang = cache.setdefault("by_language", {})
        
        # LRU eviction: if adding a new language exceeds the cap, remove the
        # entry with fewest cards (proxy for "least complete").
        if lang not in by_lang and len(by_lang) >= CACHE_MAX_LANGUAGES:
            victim = min(
                by_lang,
                key=lambda k: len(by_lang[k].get("cards", [])),
            )
            del by_lang[victim]
            _CACHE_LOGGER.info(
                "[cache] Evicted lang=%s slice (LRU, new lang=%s)", victim, lang
            )
        
        by_lang[lang] = slice_data
        
        # Size cap check
        serialized = json.dumps(cache, ensure_ascii=False)
        if len(serialized.encode("utf-8")) > CACHE_MAX_BYTES:
            _CACHE_LOGGER.warning(
                "[cache] presentation_text would exceed %d bytes for lang=%s; truncating other slices",
                CACHE_MAX_BYTES, lang,
            )
            # Keep only the current language; truncate others
            other_keys = [k for k in list(by_lang.keys()) if k != lang]
            for k in other_keys:
                del by_lang[k]
            serialized = json.dumps(cache, ensure_ascii=False)
            if len(serialized.encode("utf-8")) > CACHE_MAX_BYTES:
                # Truncate cards of the target language itself as last resort
                if slice_data.get("cards"):
                    slice_data["cards"] = slice_data["cards"][:5]
                _CACHE_LOGGER.warning(
                    "[cache] session_cache_truncated: cards truncated to 5 for lang=%s", lang
                )
        
        return cache

    @staticmethod
    def mark_language_stale(cache: dict, lang: str) -> dict:
        """
        Mark a specific language slice as stale (cache_version = -1 for that slice only).
        Called by PATCH /students/{id}/language to force re-generation in the new language
        without wiping other language slices.
        """
        by_lang = cache.get("by_language", {})
        if lang in by_lang:
            by_lang[lang]["cache_version"] = -1
        return cache

    @staticmethod
    def invalidate_all(cache: dict) -> dict:
        """
        Invalidate the entire cache (all languages). Called when cache_version bumps.
        Sets top-level cache_version to -1.
        """
        cache["cache_version"] = -1
        return cache
```

### 6.3 LLM Prompt Template

Reuses and extends the array-in/array-out pattern from `_translate_summaries_headings` in `teaching_router.py`:

```python
def _build_translation_system_prompt(language: str) -> str:
    from api.prompts import LANGUAGE_NAMES
    language_name = LANGUAGE_NAMES.get(language, "English")
    return (
        f"You are a math education translator. Translate each item in the JSON array to "
        f"{language_name}. Return a JSON array of the same length in the same order. "
        "Translate academic/math concept headings, book titles, and image captions. "
        "Do NOT translate: concept IDs, section numbers, math expressions, proper nouns "
        "like 'OpenStax'. Keep mathematical notation ($...$) intact. "
        "Return ONLY the JSON array, no extra text or markdown."
    )


async def _translate_batch(
    items: list[str],
    language: str,
    client: AsyncOpenAI,
    timeout: float = 30.0,
) -> list[str]:
    """
    Translate a list of strings to target language.
    Returns original list unchanged on any error (English fallback).
    Pattern: identical to _translate_summaries_headings in teaching_router.py.
    """
    system_prompt = _build_translation_system_prompt(language)
    user_prompt = json.dumps(items)
    
    try:
        resp = await client.chat.completions.create(
            model=OPENAI_MODEL_MINI,
            messages=[
                {"role": "system", "content": system_prompt},
                {"role": "user",   "content": user_prompt},
            ],
            max_tokens=1000,
            temperature=0.1,
            timeout=timeout,
        )
        raw = (resp.choices[0].message.content or "").strip()
        # Strip markdown code fences if present
        if raw.startswith("```"):
            import re
            raw = re.sub(r"^```(?:json)?\s*", "", raw)
            raw = re.sub(r"\s*```$", "", raw)
        translated = json.loads(raw)
        if isinstance(translated, list) and len(translated) == len(items):
            return [str(t) for t in translated]
        logger.warning("[translate_batch] Unexpected response shape for lang=%s", language)
    except Exception as exc:
        logger.warning("[translate_batch] Failed for lang=%s: %s", language, exc)
    return items  # English fallback


def compute_source_hash(text: str) -> str:
    """SHA-1 hex digest of the English source string."""
    import hashlib
    return hashlib.sha1(text.encode("utf-8")).hexdigest()
```

### 6.4 JSONB Update Pattern (Idempotent Merge)

```python
# In the UPDATE for a concept_chunk row, use PostgreSQL jsonb_set to merge
# without overwriting other language keys already in the column.
# Python-side: read current translations dict, merge in new keys, write back.

async def _upsert_translations(
    session: AsyncSession,
    table_name: str,          # "concept_chunks", "books", etc.
    row_id: UUID,
    col_name: str,            # "heading_translations", "title_translations", etc.
    new_entries: dict,        # {"ml": "...", "ta": "...", "en_source_hash": "..."}
) -> None:
    """Merge new translation entries into an existing JSONB column."""
    # Read current value
    result = await session.execute(
        text(f"SELECT {col_name} FROM {table_name} WHERE id = :id"),
        {"id": str(row_id)},
    )
    current = result.scalar_one_or_none() or {}
    current.update(new_entries)
    await session.execute(
        text(f"UPDATE {table_name} SET {col_name} = :val WHERE id = :id"),
        {"val": json.dumps(current, ensure_ascii=False), "id": str(row_id)},
    )
```

 * Primary title source is always the backend's translated concept_title field.
 * This function is a last-resort fallback for cases where no backend title is available.
 *
 * @param {string} conceptId
 * @param {function|null} t - i18next translation function (optional).
 *   If provided, localizes the "Section N.N" pattern via t("map.sectionShort", {num}).
 *   If absent, falls back to the English string "Section N.N".
 */
export function formatConceptTitle(conceptId, t = null) {
  if (!conceptId) return "";

  const sectionMatch = conceptId.match(/(\d+\.\d+)$/);
  if (sectionMatch) {
    const num = sectionMatch[1];
    return t ? t("map.sectionShort", { num, defaultValue: `Section ${num}` }) : `Section ${num}`;
  }

  // Old-style dot-separated IDs
  const parts = conceptId.split(".");
  const slug = parts[parts.length - 1] || conceptId;
  return slug
    .split("_")
    .map((word) => {
      const lower = word.toLowerCase();
      if (["to", "of", "and", "the", "a", "an", "in", "on", "for", "with"].includes(lower)) {
        return lower;
      }
      return word.charAt(0).toUpperCase() + word.slice(1).toLowerCase();

    .join(" ")
    .replace(/^\w/, (c) => c.toUpperCase());
}

## 9. CI Locale Check Script

```javascript
#!/usr/bin/env node
/**
 * Locale parity check — fails with exit code 1 if any non-EN locale
 * has keys missing from or extra to en.json.
 * 
 * Usage: node scripts/check-locales.mjs
 * Wired into: npm run lint:locales
 */

import { readFileSync, readdirSync } from "fs";
import { join, dirname } from "path";
import { fileURLToPath } from "url";

const __dirname = dirname(fileURLToPath(import.meta.url));
const LOCALES_DIR = join(__dirname, "../src/locales");

const enRaw = readFileSync(join(LOCALES_DIR, "en.json"), "utf8");
const enKeys = new Set(Object.keys(JSON.parse(enRaw)));

let exitCode = 0;

for (const fname of readdirSync(LOCALES_DIR)) {
  if (!fname.endsWith(".json") || fname === "en.json") continue;
  const lang = fname.replace(".json", "");
  const raw = readFileSync(join(LOCALES_DIR, fname), "utf8");
  const keys = new Set(Object.keys(JSON.parse(raw)));

  const missing = [...enKeys].filter((k) => !keys.has(k));
  const extra   = [...keys].filter((k) => !enKeys.has(k));

  if (missing.length > 0) {
    console.error(`[${lang}] MISSING ${missing.length} keys:`);
    missing.forEach((k) => console.error(`  - ${k}`));
    exitCode = 1;
  }
  if (extra.length > 0) {
    console.warn(`[${lang}] EXTRA ${extra.length} keys (non-fatal):`);
    extra.forEach((k) => console.warn(`  + ${k}`));
  }
}

if (exitCode === 0) {
  console.log("All locale files are in sync with en.json.");
}
process.exit(exitCode);
```

## 13. Error Handling and Resilience

| Scenario | Handling |
|----------|---------|
| `json.loads(presentation_text)` fails | `json-repair` fallback; if still fails, return `{"cache_version": -1, "by_language": {}}` and log ERROR |

## Key Decisions Requiring Stakeholder Input

1. **`processed` field removal from `GET /api/v1/books`** — the spec drops it from the new response. If any other API consumer reads `processed`, this is a breaking change. Confirm no external consumer relies on it. **Closeout decision (2026-04-24):** keep `processed: True` in the response — backward-compatible, no caller broken, removal can ship in a future PR.
2. **`lint:locales` replaces `lint` or extends it** — the spec wires `lint:locales` into `npm run lint` using `&&`. This means `eslint` failures block locale checks from running. Confirm this ordering is acceptable, or separate them in CI.
3. **`presentation_text` cleanup cron timing** — 7-day post-completion purge is proposed. Confirm this does not break any admin or analytics queries that read `presentation_text` for completed sessions.

## 14.5 Frontend wiring — Dashboard Recent Activity (added 2026-04-24)

The new `LATERAL` join in `get_student_sessions` ([teaching_router.py:1995](backend/src/api/teaching_router.py#L1995)) returns `concept_title` and `book_title` per session row in the student's preferred language. The Dashboard must consume both:

| Site | Fallback chain |
|------|----------------|
| `DashboardPage.jsx:192` (Resume CTA) | `resumeSession.concept_title || formatConceptTitle(resumeSession.concept_id, t) || t("dashboard.continueLearning")` |
| `DashboardPage.jsx:283` (subjects panel last-session) | `lastSession.concept_title || formatConceptTitle(lastSession.concept_id, t)` |
| `DashboardPage.jsx:362` (Recent Activity row) | `session.concept_title || formatConceptTitle(session.concept_id, t) || t("dashboard.studySession")` |

`book_title` does **not** require a wiring change at the Dashboard layer: `displayBookTitle(slug)` reads `bookMap[slug].title`, and `bookMap` is populated from `getAvailableBooks()` whose response is now language-resolved by the i18n overlay.

`formatConceptTitle()` remains as the last-resort fallback — it triggers only for sessions whose `concept_chunks` row is missing (orphaned chunks after content edits) or for the `_N.N`-style concept-id slugs.
